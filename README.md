# activity-normalizer

A simple CLI tool that allows you to sync intervals.icu activity names, gear, types, and descriptions back to their original data platforms.

This allows you to manually curate activity metadata in intervals.icu or Strava, and automatically sync back changes to other platforms.

## Supported Platforms

### Sources
- Intervals.icu

### Destinations
- Garmin Connect
- RidewithGPS

## CLI Offline vs. Online Mode

Activity Normalizer has mode switches for each supported platform--in online mode, it attempts to use the official platform API to make updates directly. In offline mode, it outputs a {platform}-changes.json file which can be used to actually perform the changes using other mechanisms, such as the userscripts checked in to this repo.

Official platform APIs range from "incomplete" to "actively hostile", so offline mode is generally speaking a better bet.

## Usage (CLI)

1. Install the project using `uv`
2. Configure environemnt variables (you can copy the .env.example that is checked in)
3. Run the script. Example: `uv run python main.py --days 120 --dry-run --offline-garmin --offline-rwgps`

## Usage (Userscripts)

1. Install the userscript into your userscript manager of choice
2. Navigate to the web client for the platform you wish to sync to
3. Upload the {platform}-changes.json file generated by the CLI
